# AD-001: AI or Not? — System Design

**Date**: 2026-03-29
**Author**: Brock Webb
**Status**: Draft

## 1. Problem Statement

The Scouting America AI Merit Badge (requirements 2d and 5) requires Scouts to play an "AI or Not?" game and learn about deepfakes. Existing resources are either commercial APIs, static quizzes with no data collection, or academic benchmarks unsuited for youth education. No open-source, forkable, data-collecting game exists that:

- Works as a self-paced web app AND can be projected for group discussion
- Collects anonymous response data for research and feedback
- Provides immediate visual feedback (score, percentile, item difficulty)
- Supports a growing content library via community contributions
- Includes an ML/statistics research layer for analysis

## 2. System Components

### 2.1 Frontend — The Game

Single HTML file with inline CSS and JS. No build step, no dependencies beyond what's available via CDN (if anything — prefer zero CDN deps).

**Game Flow:**
1. Landing page: title, brief explanation, "Play" button
2. Game loop (10 rounds):
   - Display media (image or embedded video)
   - Player selects "AI" or "Not AI"
   - Optional: text field "Why do you think so?"
   - Reveal correct answer with explanation
   - Brief animation/feedback per answer
3. Results screen:
   - Final score (X/10)
   - Percentile vs. all previous players ("You beat N% of players")
   - Per-item breakdown: what you guessed, what was correct, how hard each item is (% of players who got it wrong)
   - Simple bar chart: accuracy distribution across all players
   - Which items fool people the most (top 3 trickiest)

**Content Loading:**
- `content.json` loaded at page init
- 10 items randomly sampled (could be stratified by difficulty later)
- Items display in randomized order per session

**Data Submission:**
- On session complete, POST to Google Apps Script endpoint
- Payload: `{ session_id, timestamp, items: [{item_id, guess, correct, reasoning}], score }`
- On POST success, GET aggregate stats for results screen rendering

### 2.2 Backend — Google Apps Script + Sheets

**Apps Script endpoints:**
- `POST /submit` — append session data to Sheet 1, update aggregates in Sheet 2
- `GET /stats` — return current aggregate stats (item-level accuracy, score distribution, total sessions)

**Sheet 1 — Raw Sessions:**
| Column | Type | Description |
|--------|------|-------------|
| session_id | string | UUID generated client-side |
| timestamp | ISO 8601 | When session completed |
| items_json | string | JSON array of {item_id, guess, correct, reasoning} |
| score | integer | Number correct |
| total | integer | Number of items (always 10 for now) |

**Sheet 2 — Item Aggregates:**
| Column | Type | Description |
|--------|------|-------------|
| item_id | string | Matches content.json |
| times_shown | integer | How many times this item appeared |
| times_correct | integer | How many times players guessed correctly |
| accuracy | float | times_correct / times_shown |

**Sheet 3 — Score Distribution:**
| Column | Type | Description |
|--------|------|-------------|
| score | integer | 0-10 |
| count | integer | Number of sessions with this score |

### 2.3 Content Library — `content.json`

```json
{
  "version": "1.0.0",
  "meta": {
    "description": "AI or Not? Game Content Library",
    "license": "See individual item licenses",
    "curator": "brockwebb",
    "last_updated": "2026-03-29"
  },
  "items": [
    {
      "id": "img-001",
      "media_type": "image",
      "url": "https://example.com/image.jpg",
      "is_ai": true,
      "source": "Midjourney v6",
      "attribution": "Generated by @user, CC-BY-4.0",
      "license": "CC-BY-4.0",
      "explanation": "This image was generated by Midjourney. Notice the slightly unnatural lighting on the face and the blurred background text.",
      "category": "person",
      "generation_method": "midjourney-v6",
      "prior_difficulty": 0.65,
      "tags": ["face", "portrait", "photorealistic"]
    }
  ]
}
```

**Content categories** (initial): person, animal, landscape, object, scene, text, video
**Generation methods** (tracked): midjourney, dall-e, stable-diffusion, flux, sora, runway, real/camera

### 2.4 Data Export — JSONL

Periodic export from Google Sheet to `data/sessions.jsonl`. One line per session:

```json
{"session_id":"abc123","timestamp":"2026-04-08T14:30:00Z","items":[{"item_id":"img-001","guess":true,"correct":true,"reasoning":"hands looked weird"}],"score":7,"total":10}
```

### 2.5 Analysis Workbench

Python scripts in `analysis/`. Not user-facing. Read from JSONL exports.

**Planned analyses:**
1. **Bayesian difficulty updating** — Beta(α, β) prior per item from curator's `prior_difficulty`. Update: α += correct_count, β += incorrect_count. Compare posterior mean to prior.
2. **IRT 1PL** — Rasch model. Estimate item difficulty and player ability simultaneously. Requires `py-irt` or custom scipy optimization.
3. **Drift detection** — Rolling window accuracy over time. Simple changepoint detection (e.g., CUSUM or Bayesian Online Changepoint Detection). Tests hypothesis that AI-generated media gets harder to detect over time.
4. **Calibration plot** — Scatter: curator prior difficulty (x) vs. observed difficulty (y). Diagonal = perfect calibration. Measures curator's beginner's-mind accuracy.
5. **Reasoning text analysis** — Batch LLM processing to extract and cluster detection strategies. Store structured results alongside raw text.

## 3. Deployment

- Frontend: GitHub Pages from `main` branch (or `/docs` folder)
- Backend: Google Apps Script deployed as web app (anyone can access, execute as deployer)
- Fork workflow: fork repo, deploy own Apps Script, update config.js with new endpoint URL, deploy to own GitHub Pages

## 4. Content Safety

- All content in `content.json` is manually reviewed before merge
- No NSFW content, no graphic violence, no content inappropriate for ages 6+
- PRs adding content require maintainer approval
- The application renders ONLY items present in `content.json` — no user-uploaded content at runtime
